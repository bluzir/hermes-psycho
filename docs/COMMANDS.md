# Команды — шпаргалка

Ты не запоминаешь синтаксис — просто пишешь боту своими словами. Агент по фразе выбирает
протокол через [`knowledge/RESOLVER.md`](../knowledge/RESOLVER.md). Ниже — какая формулировка
какой протокол запускает, плюс служебные слэш-команды уровня фреймворка.

---

## Глаголы пользователя → протокол

| Ты пишешь (примеры) | Протокол | Что делает |
|---|---|---|
| «расскажи о себе», «хочу рассказать о себе», «составь мою карту» | [life-interview](../protocols/life-interview.md) | онбординг по блокам → наполняет `self/map.md` + `self/spheres/` + карточки людей |
| «разбери разговор с X», «вот как всё прошло» (+ голосовое) | [process-conversation](../protocols/process-conversation.md) | сохраняет транскрипт → разбор → обновляет `map/commitments/next` |
| «прогони через оптики», «почему это снова происходит», «посмотри с разных углов» | [multi-lens-read](../protocols/multi-lens-read.md) | 2–3 живых оптики + вклад самого пользователя + синтез |
| «покажи паттерны», «что у меня повторяется» | [pattern-review](../protocols/pattern-review.md) | 2–4 повторяющихся темы across базы как гипотезы; пусто → честно «нет данных» |
| «памятка по X», «скоро разговор с X», «завтра встреча» | [pre-conversation-memo](../protocols/pre-conversation-memo.md) | памятка в Telegram ≤15 строк перед разговором |
| «обзор недели», «как дела по всем» | [weekly-review](../protocols/weekly-review.md) | одно сообщение: движется / внимание / просроченное / на неделю |
| «добавь контекст», «запиши про X что…» (+ голосовое) | [add-context](../protocols/add-context.md) | новый факт о человеке → `people/<slug>/notes.md`, при значимости `profile/map` |
| «помоги решить», «сходиться / расходиться», «брать ли в кофаундеры» | [decision-support](../protocols/decision-support.md) | со-мышление: углы + вопросы, не вердикт |

Полный реестр протоколов и цепочки — [`../protocols/00-index.md`](../protocols/00-index.md).
`red-flags` работает **всегда**, поверх любого протокола — отдельного запроса не требует.

## Служебные слэш-команды бота (эксплуатация)

Это команды уровня фреймворка Hermes — набираются в чат боту. Подробности и когда они нужны —
в ранбуке [`../deploy/relationship/README.md`](../deploy/relationship/README.md).

| Команда | Что делает |
|---|---|
| `/restart` | перезапустить профиль relationship (роняет только его, не весь контейнер) |
| `/reload-skills` | пересканировать каталог скилов после правки `SKILL.md` |
| `/reload-mcp` | перечитать `mcp_servers` после изменения `config.yaml` |
| `/reset` | сбросить текущую сессию и авто-список навыков в системном промпте |
| `/new` | начать новый разговор (тоже обновляет авто-список навыков) |

## Про `/patterns` — честно

Раньше `/patterns` был граблями: gbrain-ночной `/patterns` (dream-цикл над reflections) в этом
privacy-first профиле выключен **by design** (`synthesis.dream_enabled/think_enabled: false`,
`corpus: public_only`), а если набрать `/patterns` руками — фреймворк не знал такой команды, и
модель на пустой базе **сочиняла** «gbrain-образный» ответ с фейковыми опорами и `#хэшами`.

Теперь это закрыто в профиле:
- Профиль **регистрирует свой скил** [`skills/patterns/SKILL.md`](../skills/patterns/SKILL.md),
  поэтому `/patterns` детерминированно запускает протокол
  [pattern-review](../protocols/pattern-review.md) (прогон через
  [multi-lens-read](../protocols/multi-lens-read.md)) — ровно то же, что фраза **«покажи паттерны»**.
- При пустой базе (`self/` + `people/` — только шаблоны) он **честно** говорит «данных пока нет,
  давай онбординг», а не выдумывает опоры/цитаты/`REDACTED`-заглушки.
- Ночной gbrain-синтез остаётся выключен и на уровне самого gbrain (`gbrain-config.sh` гасит
  `dream.patterns.enabled`), чтобы ничего не молотило по расписанию.

То есть и `/patterns`, и «покажи паттерны» ведут в один честный протокол. Остальные слэш-команды
ниже — служебные, уровня фреймворка.
