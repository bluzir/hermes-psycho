# Сценарии — сквозные прогоны

Четыре типовых потока в форме «ты пишешь боту → агент делает → отвечает».
У каждого — протокол, который его ведёт, и артефакт из `examples/`, на который можно посмотреть,
чтобы увидеть, как выглядит результат на «живой» (но выдуманной) системе.

> Пример в `examples/` — вымышленный: пользователь **Alex**, партнёр **Ким**. Реальных данных нет.
> Пока `self/` и `people/` не наполнены через онбординг ([life-interview](../protocols/life-interview.md)),
> разборы будут generic — агенту не с чем сверяться.

---

## Сценарий A — голосовое после свидания

**Протокол:** [`process-conversation`](../protocols/process-conversation.md) ·
**Артефакт:** [`examples/people/kim/conversations/2026-05-12-разговор-о-планах.md`](../examples/people/kim/conversations/2026-05-12-разговор-о-планах.md)

**Ты пишешь:** голосовое боту — «вот как прошёл разговор с Кимом про переезд, он опять закрылся».

**Агент делает:**
1. STT расшифровывает голос в текст.
2. Определяет человека (`kim` по `people/_index.md`), сохраняет транскрипт в
   `people/kim/conversations/YYYY-MM-DD-<тема>.md` — **до** любого разбора.
3. brain-first: читает `RESOLVER` → `knowledge/communication/` (Gottman/NVC) + `knowledge/systemic/`.
4. Называет паттерн конкретно (тест «ты вообще хочешь или тянешь?» → stonewalling у партнёра;
   репэйр «мне тревожно, когда ты замолкаешь» = NVC-наблюдение+чувство) и **вклад самого Alex**.
5. Обновляет `people/kim/map.md` (текущая динамика), `commitments.md` (кто/что/срок из разговора),
   `next.md` (что сделать до выходных).

**Агент пишет** (выжимка ≤10 строк): TL;DR + 1–2 наблюдения + один следующий шаг.
Готовый разбор — в файле разговора; посмотри его в `examples/` как эталон формата.

## Сценарий B — «прогони через оптики»

**Протокол:** [`multi-lens-read`](../protocols/multi-lens-read.md) ·
**Артефакт:** [`examples/self/map.md`](../examples/self/map.md) + [`examples/people/kim/map.md`](../examples/people/kim/map.md)

**Ты пишешь:** «почему я снова начинаю давить, когда Ким замолкает? прогони через оптики».

**Агент делает:** brain-first (`RESOLVER` → 2–3 живых слоя), читает `self/map.md` (карта Alex:
тревожный стиль, схема покинутости, часть «Тревожный сканер») и `people/kim/map.md`. Разбирает
ситуацию через выбранные оптики — не формально «все шесть», а те, что реально её освещают.

**Агент пишет** — вот как выглядит *хороший* multi-lens, планка качества:

```
🔍 kim — мульти-оптика

[Attachment]
Что видно через этот угол:
• Пауза Кима считывается тревожной системой Alex как угроза разрыва → напор,
  чтобы вернуть контакт (окно толерантности сужается).
• У Кима это включает избегающий отклик: чем сильнее запрос близости, тем дальше уход.
Вклад Alex (обязательно): третий заход за неделю — это не запрос, а проверка;
  давление, которое сам же потом читает как «он холодеет».

[Schema-therapy]
Что видно через этот угол:
• Схема покинутости: «если не напомнить о себе — забудут». Follow-up тут не про переезд,
  а про регуляцию страха, что тебя оставят.
Вклад Alex (обязательно): тема переезда стала контейнером для тревоги;
  решается не переезд, а «ты ещё здесь?».

Синтез: сквозной паттерн — близость Alex ищет через контроль темпа партнёра;
Кима это отодвигает, что подтверждает страх — петля. Ключевой вопрос: что если пауза Кима
= его способ остаться, а не уйти? Попробуй один раз назвать тревогу вслух, не двигая тему.

Хочешь обновить map.md по этому паттерну?
```

Планка (чем это лучше generic-заполнителя): **2–3 оптики, не все подряд**; каждая опирается
на конкретную запись в `self/map.md`, а не на общие слова; в **каждой** оптике назван
**вклад самого пользователя**, мягко и по делу; синтез даёт один свежий угол и одно действие,
а не вердикт. Generic-ответ («тебе стоит больше общаться и слышать друг друга») — это анти-планка:
никакой опоры на базу, ноль про вклад Alex, ничего конкретного.

## Сценарий C — «скоро сложный разговор с X»

**Протокол:** [`pre-conversation-memo`](../protocols/pre-conversation-memo.md) ·
**Артефакт:** [`examples/people/kim/next.md`](../examples/people/kim/next.md) + [`commitments.md`](../examples/people/kim/commitments.md)

**Ты пишешь:** «в выходные возвращаемся к теме переезда с Кимом — дай памятку».

**Агент делает:** читает `people/kim/map.md` (динамика, триггеры), `next.md` (что планировалось),
`commitments.md` (что кто обещал), последние 1–2 `conversations/`, и `knowledge/communication/`
(один практический ход по теме).

**Агент пишет** — памятку в Telegram, ≤15 строк, читается за 2 минуты:

```
📋 kim — переезд (романтик)

ГДЕ МЫ: тревожно-избегающая петля; тема переезда третий заход, партнёр просил темп.
ПОСЛЕДНИЙ КОНТАКТ: 12.05 — закрылся на «тянешь?», ты сделал репэйр, договорились на выходные.
ЦЕЛЬ: услышать реальную готовность Кима, а не додавить решение.
ОТКРЫТЫЕ ТЕМЫ:
• Что для Кима значит «переварить» — сколько это по времени.
• Он скидывал ок-ные районы «на неделе» — было?
КАК ГОВОРИТЬ:
• Начни с наблюдения+чувства (NVC), не с «ты снова тянешь».
• Не превращай в тест на верность; один вопрос — одна пауза, дай ответить.
```

## Сценарий D — воскресный обзор недели

**Протокол:** [`weekly-review`](../protocols/weekly-review.md) ·
**Артефакт:** все `map.md` / `commitments.md` / `next.md` (см. `examples/people/kim/*`)

**Ты пишешь:** «обзор» (или срабатывает крон в воскресенье вечером).

**Агент делает:** собирает список активных людей из `people/` + `contexts/`, по каждому читает
`map.md` / `next.md` / `commitments.md`, сортирует по срочности, вытаскивает просроченные обещания
и мета-паттерн, если одно и то же всплывает в 2+ связях.

**Агент пишет** — одно сообщение:

```
📊 Неделя 2026-05-17

В ДВИЖЕНИИ:
• kim — сделал репэйр после «тянешь?», договорились вернуться к переезду.

ВНИМАНИЕ:
• kim — инициатива будущего снова на тебе; партнёр в роли реагирующего.
  → предложение: на этот раз спроси про его темп, не предлагай свой.

ПРОСРОЧЕННЫЕ ОБЕЩАНИЯ:
• kim — «скинуть ок-ные районы на неделе» — если не пришло, мягко напомни.

НА НЕДЕЛЮ:
• kim — разговор о переезде на выходных (см. памятку).

МЕТА: замечаю — контроль темпа близости всплывает и с Кимом, и в дружбе. Тебе это откликается?
```

Если всё хорошо — агент так и пишет, не выдумывая проблемы. Если база пустая — честно предлагает
сперва пройти онбординг.
