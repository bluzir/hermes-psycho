# RESOLVER — куда филить страницу и какую оптику применять

**Читай ПЕРЕД созданием любой страницы и перед разбором любой ситуации.**
Каждое знание имеет ровно один дом (MECE). Каждая ситуация — одну или несколько оптик.

---

## Часть 1 — Таблица маршрутизации ситуаций

| Ситуация | Тип | Оптика (knowledge/) | Протокол |
|---|---|---|---|
| Разбор разговора / конфликта | любой | `communication/` + `systemic/` | `process-conversation` |
| «Меня триггернуло», ревность, тревога в близости | romantic | `attachment/` + `ifs/` | `multi-lens-read` |
| Повторяющийся паттерн в связи | любой | `schema-therapy/` + `systemic/` | `multi-lens-read` |
| Обещания, симметрия, «кто что должен» | любой | `systemic/` | `commitment-track` |
| Перед важным разговором | любой | `communication/` | `pre-conversation-memo` |
| Брать ли в кофаундеры / роли / equity | cofounder | `cofounder/` | `cofounder-diligence` *(Фаза 2; пока — `decision-support`)* |
| Сходиться/расходиться, большое решение | любой | все слои | `decision-support` |
| Абьюз, токсичность, опасность | любой | — | `red-flags` |
| Дистанция, уход в себя, «он(а) стал(а) холоднее» | romantic / friend | `attachment/` + `communication/` | `multi-lens-read` |
| Роли не совпадают с ожиданиями в паре / команде | любой | `systemic/` + `schema-therapy/` | `commitment-track` |
| Родитель триггерит, конфликт/вина/долг перед мамой/папой | family | `family/` + `ifs/` | `multi-lens-read` |
| Сепарация, границы со взрослыми родителями | family | `family/` (differentiation, boundaries) | `decision-support` |
| «Откуда у меня этот паттерн», корни в детстве | любой | `family/` + `schema-therapy/` + `attachment/` | `multi-lens-read` |
| Сливаю дела/сделки на пике успеха, не довожу до конца, прокрастинирую уязвимый шаг | любой | `self-regulation/` (+ `attachment/`, `ifs/`) | `multi-lens-read` |
| Рассказать о себе подробно / онбординг / составить карту | любой | все слои | `life-interview` |
| «Покажи паттерны / что у меня повторяется» (по всей базе) | любой | `attachment/` + `ifs/` + `schema-therapy/` + `systemic/` | `pattern-review` (+ `multi-lens-read`) |

### Как применять таблицу

1. Определи тип отношений (romantic / friend / cofounder / mixed).
2. Найди строку по ситуации. Если точного матча нет — смотри на Оптику ближайших строк.
3. Прочитай указанные страницы `knowledge/` → запусти протокол.
4. Если ситуация подпадает под несколько строк — объедини оптики, начни с более специфичной.

## Часть 2 — Куда филить новую страницу

### Дерево решений

1. Про стиль привязанности, триггер, регуляцию или репэйр в близости → `attachment/`
2. Про внутренние части, субличности в конфликте или близости → `ifs/`
3. Про схему (ранняя дезадаптивная), режим, копинг-стиль → `schema-therapy/`
4. Про коммуникацию, разрушители диалога, навыки восстановления → `communication/`
5. Про роли, системные петли, границы, баланс власти/agency → `systemic/`
6. Про родителей, семью происхождения, сепарацию, межпоколенческие паттерны → `family/`
7. Про саморегуляцию, самосаботаж, прокрастинацию, страх отказа — про **собственное действие/успех**, не про связь с другим → `self-regulation/`
8. Про equity, вестинг, роли в основании, конфликты кофаундеров → `cofounder/` *(Фаза 2)*
9. Типовая ситуация с набором ходов (сложный разговор, застревание) → `playbooks/` *(Фаза 3)*
10. Ничего из выше → `knowledge/inbox/` (+ пометка «схема требует эволюции»)

### Описание слоёв

| Слой | Что внутри | Тип отношений |
|---|---|---|
| `attachment/` | Стили привязанности (тревожный, избегающий, дезорг., надёжный); триггеры; репэйр после разрыва контакта; окно толерантности | romantic, friend |
| `ifs/` | Части, которые включаются в близости и конфликте: менеджеры, пожарные, изгнанники; протокол 6F; интеграция с режимами schema-therapy | romantic, friend, cofounder |
| `schema-therapy/` | Ранние дезадаптивные схемы (18); режимы; копинг-стили (капитуляция, избегание, гиперкомпенсация); концептуализация в отношениях | любой |
| `communication/` | Gottman: 4 всадника, bids, repair attempts; NVC (наблюдение / чувство / потребность / просьба); паттерны дистанции/близости; как просить и слышать | любой |
| `systemic/` | Роли (формальные и неявные); обратные петли; границы и enmeshment; баланс власти/agency; симметрия и асимметрия вклада | любой |
| `family/` | Семья происхождения; родительские послания, интроекты; дифференциация (Боуэн); слияние, cutoff, треугольники; репарентинг; границы со взрослыми родителями | family / origin |
| `self-regulation/` | Саморегуляция вокруг собственного действия/успеха: Upper Limit (саботаж на пике), прокрастинация как регуляция эмоций, чувствительность к отказу | любой (острее — cofounder/бизнес) |
| `cofounder/` | Equity, вестинг, роли, конфликт-сценарии при разрыве | cofounder | *(Фаза 2, ещё не создан)* |
| `playbooks/` | Типовые сложные ситуации с набором ходов — самопополняются | любой | *(Фаза 3, ещё не создан)* |

### Дедуп (обязательно перед CREATE)

1. `grep -ril "<тема>" knowledge/<layer>/` (с вариантами написания / алиасами).
2. Нашёл → UPDATE существующей страницы (добавь алиас, если нашёл под другим именем).
3. Не нашёл → CREATE по шаблону `knowledge/templates/page.md`.

### Дизамбигуация

- **Attachment vs Schema-therapy:** триггер в конкретном контакте (здесь и сейчас) → `attachment/`; устойчивый ранний паттерн (сформирован в детстве, воспроизводится везде) → `schema-therapy/`.
- **IFS vs Schema-therapy:** «часть, которая злится / боится / прячется» (голос) → `ifs/`; «схема покинутости / дефектности» (убеждение) → `schema-therapy/`. Оба могут описывать одно явление — указывай `## Связи` на оба.
- **Communication vs Systemic:** навык диалога (как говорить) → `communication/`; структура связи (роли, петли, власть) → `systemic/`.
- **Self-regulation vs Attachment:** уход/избегание **в связи с другим** (близость, дистанция) → `attachment/`; уход **от собственного действия/успеха** (саботаж на пике, прокрастинация, страх сделать ход) → `self-regulation/`. Один эпизод (обрыв контакта на тёплой ноте) активирует оба — указывай `## Связи` на оба.
- **Family vs Attachment vs Schema-therapy:** `family/` = про родителей и происхождение паттерна (откуда); `attachment/` = про триггер здесь-и-сейчас в близости; `schema-therapy/` = про устойчивое глубинное убеждение (что это). Один феномен может указывать на все три — указывай `## Связи` на все нужные.
- **Личные истории / реальные примеры** — НИКОГДА в `knowledge/`. Только в `people/<slug>/` (gitignored).
- **`self/` — карта пользователя** (gitignored): наполняется через `life-interview`; `self/map.md` читается в `multi-lens-read` для конкретного анализа вклада пользователя в паттерн (не догадки — реальная концептуализация). `self/` хранится рядом с `people/`, оба gitignored.
