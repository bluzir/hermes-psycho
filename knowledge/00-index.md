# knowledge/ — карта базы методологии

Общая теория без реальных имён и историй. Git-tracked, идёт в embedder.
Личное (люди, разговоры, коммитменты) — только в `people/` и `contexts/` (gitignored).

## Навигация

| Файл / каталог | Что внутри | Когда читать |
|---|---|---|
| [RESOLVER.md](RESOLVER.md) | Таблица ситуация → оптика → протокол; куда филить новую страницу | **всегда, перед любым разбором** |
| [schema.md](schema.md) | Формат страниц, frontmatter, правила дедупа, эпистемические требования | перед созданием страницы |
| [templates/](templates/00-index.md) | Шаблоны для новых страниц | перед созданием страницы |
| [_meta/](_meta/00-index.md) | Журнал правок пользователя (без PII) | при фиксации самообучения |

## Порядок чтения для агента

1. `RESOLVER.md` — определи оптику и протокол под задачу.
2. Нужные страницы `knowledge/<layer>/` — теория под оптику.
3. `protocols/<протокол>.md` — рабочая инструкция под ситуацию.
4. `people/<slug>/` — личный контекст (grep/file, не embedder).
