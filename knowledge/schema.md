# schema — конвенции страниц knowledge/

> **Правило приватности:** `knowledge/` git-tracked и идёт в embedder.
> Сюда кладётся **только общая теория** — без реальных имён, историй, PII
> и идентифицирующих деталей. Личное (люди под slug, разговоры, коммитменты) —
> только в `people/` и `contexts/` (gitignored).

## Frontmatter

```yaml
---
title: "Название страницы"
layer: attachment          # один из: attachment | ifs | schema-therapy | communication | systemic | family | self-regulation | cofounder | playbooks
applies_to:
  - romantic               # один или несколько: romantic | friend | cofounder | family | all
tags: []                   # свободные теги, kebab-case
updated: YYYY-MM-DD
---
```

## Секции тела

```
## Суть
Краткое (2–5 предложений) определение концепта или явления. Что это, откуда, в каком контексте появляется.

## Как проявляется
Поведенческие маркеры, речевые индикаторы, паттерны — то, что видно снаружи или слышно в разговоре.

## Что с этим делать
Рычаги, вмешательства, вопросы для со-мышления, техники. Без личных инструкций (они — в protocols/).

## Связи
Ссылки на связанные страницы этой базы знаний в формате [[layer/slug]].
При создании страницы — хотя бы одна связь или явная пометка «ещё нет».

## Источники
- Автор / книга / статья — краткое описание (год).
```

## Slug и размещение

- Имя файла: `kebab-case.md`, lowercase.
- Коллизия → дизамбигуатор в скобках (`dismissing-avoidant.md`, `anxious-preoccupied.md`).
- Каждый каталог `layer/` с `.md`-файлами должен иметь `00-index.md`.

## Дедуп (обязательно перед CREATE)

1. `grep -ril "<тема>" knowledge/<layer>/` — искать имя + варианты.
2. Нашёл → UPDATE существующей страницы.
3. Не нашёл → CREATE по шаблону `knowledge/templates/page.md`.

## Эпистемическая дисциплина

Каждое непустое утверждение — с источником или пометкой `[без источника]`.
Пустая секция → `[Нет данных пока]` (не опускать — это сигнал для будущего наполнения).
