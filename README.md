# hermes-psycho

Движок персонального AI-психолога. Форкаешь репозиторий — и собираешь своего
со-мыслителя под любой домен: тревога, самооценка, карьера, привычки, отношения.
В комплекте идёт готовая вертикаль **«отношения»** как рабочий пример: на ней видно,
как связаны все три слоя и как из одного манифеста разворачивается персона агента.

Это не чат-бот с промптом. Это база методологии + операционные плейбуки + персона,
которые собираются в одного агента, живущего в Telegram и работающего только на своего владельца.

---

## Что это

Три слоя, у каждого своя ответственность:

- **`knowledge/`** — методология. Общие психологические фреймворки без личных историй:
  attachment (стили привязанности), ifs (Internal Family Systems), schema-therapy,
  communication (Gottman / NVC), systemic (роли, петли, границы), family, self-regulation.
  Плюс `RESOLVER.md` — маршрутизатор «ситуация → слой → протокол», `schema.md` — формат
  страницы, `00-index.md` — карта базы.
- **`protocols/`** — операционные «шляпы» агента. Как обработать разговор, прогнать ситуацию
  через несколько оптик (`multi-lens-read`), отследить обещания, собрать weekly-review,
  среагировать на red-flags, провести life-interview. Реестр — `protocols/00-index.md`.
- **`hermes/SOUL.md`** — персона агента: роль, тон, язык, обращение. Не пишется руками —
  генерируется из `hermes-profile.yaml`.

**Публичное отделено от приватного.** В семантический индекс (gbrain) идёт только теория из
`knowledge/` — без единого личного факта. Личные записи (`people/`, `self/`, `contexts/`)
агент читает напрямую через grep/файлы; они в `.gitignore` и никогда не покидают локальную машину.

## Структура репозитория

```
hermes-psycho/
├── hermes-profile.yaml     # единый манифест профиля: owner, домен, персона, деплой
├── hermes/
│   └── SOUL.md             # персона агента (генерируется из манифеста)
├── knowledge/              # методология → семантический индекс (без PII)
│   ├── RESOLVER.md         # маршрутизатор: ситуация → оптика → протокол
│   ├── schema.md           # формат страницы, дедуп, эпистемические правила
│   ├── 00-index.md         # карта базы методологии
│   ├── attachment/         # стили привязанности, триггеры, репэйр
│   ├── ifs/                # части в близости и конфликте, протокол 6F
│   ├── schema-therapy/     # схемы, режимы, копинг-стили
│   ├── communication/      # Gottman, NVC, repair attempts
│   ├── systemic/           # роли, петли, границы, власть/agency
│   ├── family/             # семья происхождения, дифференциация, границы
│   ├── self-regulation/    # самосаботаж, прокрастинация, страх отказа
│   ├── templates/          # шаблоны новых страниц
│   └── _meta/              # журнал правок владельца (без PII)
├── protocols/              # плейбуки агента + реестр 00-index.md
├── scripts/                # рендер профиля, гейты приватности, gbrain, обслуживание
│   ├── render-hermes-profile.py
│   ├── check-hermes-profile.sh
│   ├── check-privacy.sh
│   └── check-structure.sh
├── deploy/
│   └── relationship/       # ранбук деплоя вертикали: Telegram-бот, gbrain, allowlist
├── docs/
│   └── CUSTOMIZATION.md     # гайд «собери своего AI-психолога за 5 шагов»
├── tests/                  # тесты рендера профиля
├── people/                 # ← GITIGNORED: страницы людей (в репо только _TEMPLATE)
├── self/                   # ← GITIGNORED: карта владельца (в репо только _TEMPLATE)
└── contexts/               # ← GITIGNORED: контексты разговоров (создаётся локально)
```

## Быстрый старт

1. **Форкни** репозиторий.
2. **Отредактируй `hermes-profile.yaml`** — единственный источник правды о профиле:
   владелец, домен, имя ассистента, роль и тон персоны, таймзона.
3. **Сгенерируй персону и конфиги:**
   ```bash
   python3 scripts/render-hermes-profile.py --write
   ```
   Это перепишет `hermes/SOUL.md`, `config.yaml.example` и bash-скрипты из манифеста.
4. **Проверь сборку** — один smoke-прогон без сети и контейнера:
   ```bash
   bash scripts/verify.sh
   ```
   Он гоняет render `--check`, тесты, гейт приватности и гейт структуры и печатает
   компактный OK/FAIL по шагам. Тесты отдельно — `bash scripts/test.sh` (НЕ голый
   `python3 -m unittest discover` из корня — он находит 0 тестов и «зеленеет» вхолостую).
5. **Посмотри `_TEMPLATE`-шаблоны** в `people/` и `self/` и выдуманный пример в `examples/` —
   чтобы понять, как агент хранит и читает личный контекст.
6. **Разверни** по ранбуку `deploy/relationship/README.md` (Telegram-бот, gbrain-индекс,
   single-user allowlist).

Пошаговый разбор кастомизации под свой домен — в [`docs/CUSTOMIZATION.md`](docs/CUSTOMIZATION.md).
Сквозные сценарии — [`docs/SCENARIOS.md`](docs/SCENARIOS.md);
шпаргалка команд — [`docs/COMMANDS.md`](docs/COMMANDS.md);
устройство движка — [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

### Локально vs на сервере

Частый вопрос: «а где веб-приложение / какой порт открыть?». **Локально запускать нечего** —
это контент/конфиг-пак, а не сервер. Локально ты только **авторишь** (правишь манифест и
`knowledge/`/`protocols/`) и **валидируешь** (`render` → `scripts/verify.sh`). Ни порта, ни UI,
ни `dev server`. **Запуск = деплой:** по ранбуку `deploy/relationship/README.md` пак разворачивается
в контейнер внешнего фреймворка Hermes, и ты общаешься с ним через своего Telegram-бота.
Подробнее — [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

### Онбординг / первый запуск

После деплоя система пустая: `self/` и `people/` — только шаблоны, агенту не с чем сверяться,
и разборы будут generic. Чтобы наполнить её — первое сообщение боту: **«хочу рассказать о себе»**.
Это запускает протокол [life-interview](protocols/life-interview.md): агент по блокам (происхождение,
семья, романтика, бизнес, друзья) задаёт по 2–3 вопроса за раз, после каждого блока пишет выжимку
в `self/map.md` и `self/spheres/`, заводит карточки людей в `people/<slug>/`. Прервать можно в любой
момент — прогресс в `self/interview-progress.md`, при возврате продолжит с незаконченного блока.
Пока карта не собрана, разборы (`multi-lens-read`, `pattern-review`) честно скажут, что данных ещё нет.

## Не только про отношения

Вертикаль «отношения» — это пример, а не потолок. Ядро переиспользуется под другие домены
почти без изменений, потому что базовые слои — общие психологические фреймворки, не привязанные
к теме близости:

- **Тревога / самооценка** — attachment, ifs и schema-therapy напрямую описывают триггеры,
  внутренние части и глубинные схемы, которые крутятся вокруг любого стрессора, а не только
  отношений.
- **Карьера / прокрастинация** — self-regulation (Upper Limit, страх отказа, прокрастинация как
  регуляция эмоций) и systemic (роли, границы, баланс agency) работают на рабочем контексте так же,
  как на личном.
- **Привычки** — self-regulation + communication (как договариваться с собой и близкими) дают
  каркас без единой строки про «пару».

Меняешь домен в манифесте, оставляешь универсальные слои, добавляешь один-два своих по формату
`knowledge/schema.md`, обновляешь `knowledge/RESOLVER.md` — и тот же движок работает на новую тему.

## Приватность

Приватность заложена в архитектуру, а не прикручена сбоку:

- `people/`, `self/`, `contexts/`, `data/`, аудио и транскрипты — в `.gitignore`. В публичном
  репозитории лежат только `_TEMPLATE`-шаблоны и выдуманный `examples/`. Реальные
  данные не коммитятся никогда.
- В семантический индекс (gbrain) идёт **только** `knowledge/` — общая теория без PII. Личные
  записи агент читает через grep/файлы, они не эмбедятся.
- `scripts/check-privacy.sh` — гейт: сверяется с денилистом (`.privacy-denylist`) и проверяет,
  что личные каталоги и секреты не попали под git. Прогоняй перед каждым коммитом.
- `scripts/check-structure.sh` — проверяет, что дерево репозитория соответствует ожидаемой раскладке.

## Происхождение

Построено на фреймворке **Hermes** — персональном агент-движке (профиль, SOUL, gbrain-индекс,
Telegram-деплой, крон-ритуалы). `hermes-psycho` — это специализация Hermes под задачу
персонального AI-психолога: методология в `knowledge/`, плейбуки в `protocols/`, single-user изоляция.

---

## Дисклеймер

Это инструмент со-мышления, а не медицинская помощь. Он **не ставит диагнозов, не назначает
лечение и не заменяет живого психотерапевта или психиатра**. При остром кризисе, риске для жизни
или насилии — обращайся к специалистам и экстренным службам, а не к боту. Красные флаги (абьюз,
токсичность, суицидальные мысли) вынесены в `protocols/red-flags.md` и `protocols/crisis-support.md`
именно для того, чтобы агент их не «прорабатывал», а разворачивал к реальной помощи.
