# onboarding — первое наполнение базы

Пользователь вызвал `/onboarding` (или написал «хочу рассказать о себе» / «давай наполним базу»).
Это **профильная операция**, а не команда gbrain.

## Что делать

Выполни протокол [`protocols/onboarding.md`](../../protocols/onboarding.md) — он оркестрирует
меню из 4 путей:

1. **Интервью** — живой разговор ([life-interview](../../protocols/life-interview.md)).
2. **Транскрипты психотерапии** — выгрузки сессий.
3. **Дневники** — личные записи.
4. **Переписки с важными людьми** — экспорт чатов.

Рекомендуй **смешивать**: короткое интервью + самый насыщенный источник — так реальная карта
собирается быстрее, чем одним каналом.

Для больших выгрузок есть drop-folder `data/raw/{therapy,journals,chats}/` (см. `data/README.md`):
пользователь кладёт туда сырьё и говорит «переработай инбокс» — тогда запускается
[`ingest`](../ingest/SKILL.md). Сырьё gitignored и не попадает в индекс; в базу идут только
дистилляты в `self/` + `people/` (тоже приватные).

## Честность

Не выдумывай данные и не анализируй по пустой базе. Пока пользователь не дал реального материала —
предлагай пути наполнения, а не generic-выводы. Нет реальной опоры в `self/`/`people/` — нет темы.
