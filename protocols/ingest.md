# Протокол: инжест (bulk distiller)

**ТРИГГЕР:** «переработай инбокс», «переработай data/raw», «залей транскрипты / дневники /
переписки», приложен файл или папка с дампом. Также — маршрутизация из `onboarding`
(пути 2–4). Для одного разговора — не сюда, а в `process-conversation`.

**ВХОД:** дамп документов одного или нескольких типов — папка `data/raw/…`, вставленный текст
или приложенный файл. Агент читает то, что ему дали.
**ВЫХОД:** дистилляты в `self/map.md`, `self/spheres/*`, `self/values.md`, `people/<slug>/*` —
каждый с provenance-строкой; гипотезы помечены; ничего не выдумано — только из текста.
Отчёт пользователю: что залито, что извлечено, что дальше.
**ОПТИКА:** RESOLVER → `attachment/` + `ifs/` + `schema-therapy/` (терапия, дневники) +
`systemic/` + `communication/` (переписки) + `self-regulation/`, `family/` по содержимому.

> Инжест **исполняет агент** (Claude Code или деплой-бот). Ниже — самодостаточная инструкция.
> Красные флаги (`red-flags.md`) — ⚠️ отдельным сообщением сразу, поверх всего.

---

## Шаг 0 — Принять вход и разложить сырьё

Вход гибкий — агент читает, что дали:

- **Папка** `data/raw/therapy/`, `data/raw/journals/`, `data/raw/chats/<slug>/` — читать всё внутри.
- **Вставленный текст** — обработать как есть; при заливке сохранить в соответствующую папку `data/raw/…`.
- **Приложенный файл** — прочитать; сохранить копию в `data/raw/…`.

Определить тип источника (therapy / journals / chats) по папке или по содержимому.
Сырьё оставить в `data/raw/…` как source of truth — **не удалять**.

**Приватность (жёстко):**
- `data/raw/` — gitignored, НИКОГДА не встраивается в индекс (`import_policy: public_only` исключает `data/`).
- В дистилляты сырой текст не копировать — только выжимку.
- Каждый дистиллят несёт **provenance-строку**, напр.: `источник: therapy (локально, не в индексе)`.

## Шаг 1 — brain-first + дедуп

- `knowledge/RESOLVER.md` → оптика под тип источника.
- Прочитать существующие `self/map.md`, `self/spheres/*`, `self/values.md`, `people/*/map.md`.
- **Дедуп:** не создавать дубль. Совпало с существующей записью → UPDATE (дополнить, датировать),
  не плодить вторую страницу/раздел. Новый человек → создать карточку по `people/_TEMPLATE/`.

## Шаг 2 — Дистилляция по типу источника

### A. Транскрипты терапии → `self/map.md` + `self/values.md`

Что извлечь и куда:

| Извлекаешь | Куда |
|---|---|
| Стиль привязанности (маркеры близости/дистанции/репэйра) — **гипотеза** | `self/map.md` → Привязанность |
| Части IFS (менеджеры, пожарные, изгнанники — какие звучат) | `self/map.md` → Ключевые части (IFS) |
| Схемы (ранние дезадаптивные), режимы, копинг-стиль | `self/map.md` → Доминирующие схемы |
| Ядровые раны / болезненные темы, к которым возвращается | `self/map.md` → Семья происхождения / отдельный раздел |
| Повторяющиеся темы через сессии | `self/map.md` → Паттерны |
| Что называет важным / за что держится | `self/values.md` |

Оптика: `attachment/` + `ifs/` + `schema-therapy/` (+ `family/`, `self-regulation/` по содержимому).

### B. Дневники → `self/map.md` + `self/spheres/*`

| Извлекаешь | Куда |
|---|---|
| Повторяющиеся триггеры (что раз за разом поднимает реакцию) | `self/map.md` → Паттерны |
| Ценности (что называет важным, к чему возвращается) | `self/values.md` (+ раздел «расходится с поведением», если видно) |
| Паттерны настроения (циклы, спады/подъёмы, привязка к событиям/людям) | `self/spheres/*` соответствующей сферы |
| Паттерны по сферам (семья / романтика / дело / друзья) | `self/spheres/family.md` / `romantic.md` / `work.md` / `friends.md` |

Оптика: `schema-therapy/` + `attachment/` + `self-regulation/`.

### C. Экспорты переписок → `people/<slug>/*`

**Определить, кто важен** — по объёму (много сообщений) и эмоциональному заряду
(конфликты, близость, повторяющиеся темы). Для каждого важного человека — карточка:

| Извлекаешь | Файл |
|---|---|
| Кто это, тип связи, контекст | `people/<slug>/profile.md` (из `people/_TEMPLATE/profile.md`) |
| Динамика, привязанность, части, паттерн — **гипотезы** | `people/<slug>/map.md` (из `_TEMPLATE/map.md`) |
| Обещания (кто / что / срок) | `people/<slug>/commitments.md` (формат — `commitment-track.md`) |
| Дайджест разговоров (ключевые темы/эпизоды, без дословного дампа) | `people/<slug>/conversations/` (сводный файл или по эпизодам) |
| Прочие факты, всплывающие в переписке | `people/<slug>/notes.md` |

Добавить строку в `people/_index.md`. Оптика: `communication/` + `systemic/` + `attachment/`.

> Для разбора **одного** конкретного разговора глубже — `process-conversation`.

## Шаг 3 — Честность и провенанс

- **Только из текста.** Не изобретать привязанность, части, «опоры» или события, которых в дампе нет.
- **Гипотезы — с пометкой:** «гипотеза: …», «похоже на …», «выглядит как …». Не диагноз.
- **Provenance в каждом дистилляте:** строка вида `источник: journals (локально, не в индексе), дата инжеста YYYY-MM-DD`.
- **Дедуп перед записью** (Шаг 1) — UPDATE, а не дубль.
- Пусто по разделу → оставить `[нет данных пока]`, не досочинять.

## Шаг 4 — Отчёт пользователю и обновление прогресса

Обновить `self/interview-progress.md` (раздел **Источники** и **Покрыто / не хватает** —
формат в `onboarding.md`).

Отчёт одним сообщением:

```
Залил: <источник> (<сколько документов>). Сырьё лежит в data/raw/… (локально, не в индексе).
Извлёк: <кратко по типам — attachment/части/схемы/триггеры/люди>.
Обновил: <файлы self/ и people/>. Гипотезы помечены.
Сейчас покрыто: <сферы/люди>. Слабо: <что>. Дальше: <следующий источник или анализ>.
```

**Follow-up:** когда источников достаточно — предложить `pattern-review` (сквозные паттерны
по всей базе) или `multi-lens-read` (углубиться в один паттерн).

## Правила

- **Сырьё в `data/raw/`, gitignored, не в индексе.** Не встраивать, не удалять — source of truth.
- **Дистилляты несут provenance.** Каждая запись помечена источником и «локально, не в индексе».
- **Только из текста, гипотезы — с пометкой.** Никаких выдуманных «опор».
- **Дедуп обязателен.** Совпало → UPDATE, не дубль.
- **Без PII в `knowledge/`.** Личное — только в `self/` и `people/` (gitignored). Паттерны без имён → в `knowledge/`.
- **Красные флаги — ⚠️ немедленно** (`red-flags.md`), до любой выжимки.
- **Один разговор — не сюда:** `process-conversation`. Follow-up анализ: `multi-lens-read` / `pattern-review`.
