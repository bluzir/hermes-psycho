# Протокол: память между сессиями

Что куда сохранять, чтобы важное не терялось и всплывало в нужный момент.
Три уровня памяти — у каждого своя роль, не дублируй вслепую.

---

## 1. `session_search` — поиск по прошлым диалогам

История диалогов сохраняется автоматически (движок Hermes, SQLite+FTS5).
Переживает сброс сессии и рестарт. Это **поиск**, не место для записи.

- Зови на «помнишь, мы...», «в прошлый раз», «что мы решали по...».
- Сначала поищи — потом отвечай. «Не помню» без поиска — ошибка.
- Для смыслового поиска — `gb query` (если gbrain доступен).

---

## 2. `memory` — curated-память (USER.md / MEMORY.md)

Локальные файлы, грузятся в начало каждой новой сессии.

**Что класть:**
- **USER.md** — про пользователя: рабочие предпочтения, стиль, договорённости, как с ним удобнее работать.
- **MEMORY.md** — операционные уроки: что сработало, грабли (напр. «patch падает → write_file»), устройство окружения.

**Гигиена:** коротко и по делу (лимит ~2200 символов на MEMORY.md, ~1375 на USER.md);
`add` для нового, `replace`/`remove` для обновления; устаревшее чисти.

**Жёсткое правило:** в `memory` НИКОГДА не попадают реальные имена людей из базы,
детали их жизни, цитаты из разговоров. Это уходит в системный промпт.
Любой факт о конкретном человеке → файлы `people/<slug>/`, не в `memory`.

---

## 3. Файлы — источник истины

- Люди → `people/<slug>/` (profile.md, map.md, commitments.md, conversations/, next.md, notes.md).
- Контексты (группы) → `contexts/<slug>/context.md`.
- Предпочтения пользователя по работе → `knowledge/_meta/preferences.md` (детально, с датой) —
  **пиши СРАЗУ при правке или несогласии пользователя, в тот же ход, не откладывай**.
  Самое важное продублируй строкой в `memory` (USER.md), чтобы всплывало само.
- Теория и паттерны → `knowledge/` (без PII).
- Пробелы базы → `_gaps/queue.md`.

---

## Правило большого пальца

- Нужно **найти**, о чём говорили? → `session_search`.
- Нужно, чтобы факт **сам всплывал** (и это не про конкретного человека)? → `memory`.
- Это про **конкретного человека** или это теория? → **файл**, только файл.
