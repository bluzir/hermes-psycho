# Протокол: трекинг обещаний

**ТРИГГЕР:**
- пользователь фиксирует обещание («договорились, что он/я...», «она сказала, что...»);
- пользователь хочет проверить статус («что там с...», «мы ещё не обсудили X?»);
- `weekly-review.md` обнаружил просроченное или асимметричное — вызывает этот протокол;
- Любой разговор в `process-conversation.md` выявил новые коммитменты.

**ВХОД:** slug + новый коммитмент / запрос статуса / файл `commitments.md`.
**ВЫХОД:** обновлённый `people/<slug>/commitments.md`; при асимметрии или просрочке — сигнал пользователю.
**ОПТИКА:** RESOLVER → `systemic/` (симметрия, баланс, роли).

---

## Формат файла `commitments.md`

```markdown
# Обещания — <slug>
*Обновлено: YYYY-MM-DD*

## Активные

| Кто | Что | Срок | Статус | Источник |
|---|---|---|---|---|
| Пользователь | <обещание> | YYYY-MM-DD / «скоро» / — | 🟡 в процессе | разговор YYYY-MM-DD |
| <slug> | <обещание> | YYYY-MM-DD | 🔴 просрочено | разговор YYYY-MM-DD |

## Выполнено (архив)

| Кто | Что | Выполнено |
|---|---|---|
| Пользователь | <обещание> | YYYY-MM-DD |

## Заметки по симметрии
<!-- наблюдения: кто несёт больше, где дисбаланс, открытые вопросы -->
```

**Статусы:**
- 🟢 выполнено
- 🟡 в процессе / ждём
- 🔴 просрочено
- ⚪ открытое (без срока)

## Шаги

### 1. Новый коммитмент

1. Прочитать `people/<slug>/commitments.md` (если файл есть).
2. Добавить строку в «Активные» с заполненными полями.
3. Если срок не назван — поставить `—` и в «Источник» указать разговор.
4. Подтвердить пользователю одной строкой: «Записал: <кто> → <что> к <срок>».

### 2. Обновление статуса

1. Пользователь говорит «сделали» / «не сделал» / «перенесли» — найти строку, обновить статус.
2. 🟢 → перенести в архив «Выполнено».
3. 🔴 → оставить в «Активные», обновить срок если перенесли.

### 3. Ревизия (по запросу или из weekly-review)

1. Прочитать весь `commitments.md`.
2. Проверить каждую строку: срок прошёл → статус 🔴, сигнал пользователю.
3. **Симметрия:** посчитать, сколько обещаний у пользователя vs у другого. Заметная асимметрия —
   подсветить в «Заметки по симметрии» и упомянуть пользователю (без морализаторства).
4. Просроченные без явной причины — вынести в сообщение пользователю:
   «⚠️ Просрочено у <slug>: <что>. Актуально ещё?»

### 4. Сигнал при асимметрии

Признаки дисбаланса, которые стоит называть:
- пользователь берёт на себя большинство обещаний, другой — нет;
- Другой систематически не выполняет в срок без объяснений;
- Обещания идут только в одну сторону.

Тон — наблюдение, не приговор: «Замечаю, что последние N обещаний все на тебе —
это ок или стоит обсудить симметрию?»

---

## Правила

1. **Записывать сразу.** После любого разговора с новым обещанием — не откладывать.
2. **Дословность.** «Он сказал "скоро"» — не домысливать дату; писать `«скоро»` в поле Срок.
3. **Без PII в knowledge/.** `commitments.md` — в `people/<slug>/` (gitignored).
4. **Не теряй в архиве.** Выполненные перемещаются в «Выполнено», не удаляются — история важна.
5. **Не навязывай вердикт.** Асимметрия — сигнал для разговора, не повод решить за пользователя.
