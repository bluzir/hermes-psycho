# Протокол: добавить контекст по человеку

**ТРИГГЕР:** Пользователь прислал голос или текст о ком-то из базы — не как разговор для разбора,
а как новый факт, наблюдение, деталь: «по <slug>: узнал, что...», «кстати, он/она...»,
«запиши: <что-то>». Зонтик над `add-context` — протокол по умолчанию для любого
упоминания человека, не попавшего в другие протоколы.

**ВХОД:** текст / войс (транскрибирован Hermes) + slug или имя человека.
**ВЫХОД:** запись в `people/<slug>/notes.md` или создание новой заметки + при необходимости
обновление `profile.md` или `map.md` + подтверждение пользователю одной строкой.

---

## Алгоритм

### 1. Определить человека

- Явный slug → сразу.
- Нет slug → `grep -rl "<имя>" people/` по полю `aliases:` в `profile.md`.
- Не нашёл → один вопрос: «Кто это? Есть в базе или новый?»
  При новом человеке — сначала создать `people/<slug>/profile.md` (минимум), потом добавить контекст.

### 2. Сохранить факт

Записать в `people/<slug>/notes.md` (дописать в конец):

```markdown
## YYYY-MM-DD — <краткая тема>
*Источник: голос | текст*

<факт / наблюдение / дословно как сказал пользователь>
```

Дословность важна: слова пользователя — уже интерпретация, не трогать.

### 3. Оценить значимость

- **Мелкий факт** (деталь, контекст, логистика) → только в `notes.md`. Пользователю: «Записал: <факт>».
- **Значимый факт** (новое событие, изменение динамики, материал для map.md) →
  также обновить `people/<slug>/profile.md` или `map.md` + предложить пользователю: «Обновил профиль.
  Добавить в map.md как паттерн?» Согласие → правка.
- **Не о человеке** (мысль пользователя об отношениях в целом, наблюдение) → в `knowledge/inbox/`
  (без PII) или в `_gaps/queue.md` если это про пробел базы.

### 4. Подтвердить одной строкой

«Записал: <факт> → `people/<slug>/notes.md`»
Развёрнутый ответ — только если пользователь спросил.

---

## Правила

1. **Сначала сохранить, потом предлагать.** Заметка пишется до любых обновлений карт.
2. **Один уточняющий вопрос максимум.** Не превращать в допрос.
3. **Не интерпретировать сверх сказанного.** Факты — факты; гипотезы рождаются в разборе.
4. **Без PII в knowledge/.** Конкретный человек → только `people/` (gitignored).
5. **Красные флаги** (см. `red-flags.md`) — проверяешь в любом контексте, включая заметку.
