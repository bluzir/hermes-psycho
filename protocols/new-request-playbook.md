# Протокол: незнакомый запрос → новый плейбук

**ТРИГГЕР:** Пользователь описывает ситуацию или паттерн, которого нет в `knowledge/playbooks/`;
или явно просит: «как вообще работать с X», «как думать про ситуацию типа Y».

**ВЫХОД:** черновик плейбука в `knowledge/playbooks/draft-<имя>.md` + строка в индексе после
подтверждения пользователя.

---

## Алгоритм

### 1. Определить тип ситуации

Зафиксировать точно — словами пользователя (цитата, не пересказ): пересказ уже содержит гипотезу.
Тип: romantic / friend / cofounder + суть паттерна или ситуации.

### 2. Проверить, нет ли плейбука уже

- `grep -ril "<тема>" knowledge/playbooks/`
- Есть похожий → адаптировать под новый контекст, не дублировать.
- Нет → создавать.

### 3. Рамка по RESOLVER

- `knowledge/RESOLVER.md` → оптика → слои.
- Раскрыть через: что за потребностью / паттерном (по attachment, IFS, schema-therapy);
  какие оптики помогают думать про это; типовые ходы и чего избегать.

### 4. Черновик плейбука

Структура — обязательная:

```markdown
# Плейбук: <тип ситуации>
*Создан: YYYY-MM-DD | Статус: черновик*

## Ситуация
С чем приходит запрос, типичные формулировки.

## Как распознать
Маркеры в описании ситуации.

## Что за этим стоит (гипотезы)
Оптика (attachment / IFS / schema / systemic) + типичный паттерн.

## Углы и ходы
Конкретные вопросы и направления разбора.

## Чего не делать
Типичные ошибки именно в этом типе ситуации.

## Признаки продвижения
Как выглядит прогресс — наблюдаемо.
```

Шесть разделов — обязательный минимум.

### 5. Показать пользователю (выжимку, не файл целиком)

```
🧩 Новый плейбук: <тип ситуации>
РАМКА: <оптика + потребность/паттерн>, 2–3 строки.
КЛЮЧЕВЫЕ ХОДЫ: 2–3 вопроса / направления.
Черновик: knowledge/playbooks/draft-<имя>.md. Что поправить?
```

После правок → переименовать в `knowledge/playbooks/<имя>.md`, добавить строку в `00-index.md`.

---

## Правила

- Один плейбук = один тип ситуации, не «всё про романтику».
- Плейбук — гипотезы по умолчанию: конкретная ситуация всегда главнее шаблона.
- **Без PII.** В плейбук попадает только теория и типовые паттерны — без реальных имён.
- Если плейбук дважды «не лёг» → пересматривать рамку плейбука, не ситуацию.
