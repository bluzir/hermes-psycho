# examples/onboarding/ — как сырьё превращается в дистиллят

> **Это выдуманный пример входа для онбординга** и того, что из него дистиллируется.
> В реальном деплое сырьё (транскрипты терапии, экспорты переписок, дневники, интервью)
> лежит в `data/raw/` — этот каталог **gitignored и не в индексе**. Здесь фрагменты
> положены в `examples/` только для наглядности, чтобы было видно связь вход→выход.

## Зачем это здесь

Онбординг умеет заходить с четырёх типов источников: **интервью**, **транскрипты
терапии**, **дневники**, **переписки**. Из любого из них агент делает не копию, а
**выжимку**: несколько помеченных гипотез в `self/map.md` и карточки людей — с провенансом,
откуда взято.

## Что где посмотреть

**Сырьё (в реале — `data/raw/`, gitignored):**
- `raw-therapy-excerpt.md` — выдуманный фрагмент транскрипта сессии Alex с терапевтом.
- `raw-chat-excerpt.md` — выдуманный фрагмент переписки Alex↔Ким.

**Дистиллят (то, что агент реально записал бы):**
- `distilled-self-map-excerpt.md` — что из транскрипта попало в [[self/map]] (гипотезы + провенанс).
- `distilled-kim-card-excerpt.md` — что из переписки попало в `people/kim/` (гипотезы + провенанс).

## Правила дистилляции (видны на примере)

- **Только из текста.** Ничего не выдумывается сверх сказанного в источнике.
- **Всё помечено как гипотеза.** Формулировки терапии/схем/привязанности — предположения, не диагнозы.
- **Сырьё не копируется дословно.** В дистилляте — вывод и один короткий якорный пример, не абзацы транскрипта.
- **Провенанс обязателен.** Каждый блок несёт строку «источник: …», чтобы было видно, откуда гипотеза.
