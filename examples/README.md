# examples/ — выдуманный пример заполненной системы

> **Это НЕ реальные люди.** Alex и Ким — полностью вымышленные персонажи.
> Каталог показывает, как выглядит система, когда она «живая» и наполнена:
> карточка человека, карта динамики, обещания, разобранная беседа, само-модель.
> Здесь нет ничьих настоящих данных — только демонстрация формата.

## Легенда примера

**Alex** — пользователь системы. Тревожный стиль привязанности, схема покинутости,
внутренний «Тревожный сканер», который следит за температурой отношений.

**Ким** — партнёр Alex (романтические отношения ~1.5 года). Избегающий стиль:
при сближении уходит в работу и молчание. Классическая тревожно-избегающая ловушка.

## Что где посмотреть

- `people/kim/profile.md` — кто такой Ким, контекст связи.
- `people/kim/map.md` — концептуализация динамики через attachment / IFS / schema-therapy.
- `people/kim/commitments.md` — обещания с дедлайнами и флагом симметрии.
- `people/kim/next.md` — что Alex стоит сделать до следующего контакта.
- `people/kim/conversations/2026-05-12-разговор-о-планах.md` — разобранная беседа.
- `self/map.md` — само-модель Alex.
- `self/spheres/romantic.md` — как паттерны Alex проявляются в романтике.

Чтобы начать своё — копируй `_TEMPLATE`-каталоги из `people/`, `self/`, `contexts/`,
а этот пример держи как ориентир.
