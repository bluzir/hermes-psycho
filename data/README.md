# data/ — drop-folder для онбординга

Сюда кладётся **сырьё** для наполнения базы. Раскладка:

```
data/raw/
  therapy/    # транскрипты психотерапии
  journals/   # дневники, личные записи
  chats/      # переписки с важными людьми (экспорт чатов)
```

## Что важно знать

- **Всё в `data/` — приватное и gitignored.** Отслеживается git-ом только этот `data/README.md`;
  всё остальное (включая `data/raw/…`) игнорируется и никогда не коммитится.
- **Сырьё НЕ попадает в индекс.** Папки `data/raw/…` не импортируются в gbrain — они остаются
  только на диске, читаются grep/file при переработке.
- **В базу идут только дистилляты.** Из сырья извлекаются структурированные записи в `self/`
  (о тебе) и `people/` (о людях и связях), с провенансом (из какого источника). Эти дистилляты
  тоже приватные (local_private) и в git/gbrain не уходят.

## Как запустить переработку

Положи файлы в нужную подпапку `data/raw/…` и напиши **«переработай инбокс»** — запустится
протокол `protocols/ingest.md` (скилл `ingest`), который разберёт сырьё и разложит дистилляты
по `self/` + `people/`.

Первое наполнение проще начать с онбординга: напиши «хочу рассказать о себе» или `/onboarding` —
агент предложит 4 пути (интервью / транскрипты психотерапии / дневники / переписки) и подскажет,
что удобнее смешать.
